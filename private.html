Bug Bounty Severity Chart

Priority & severity definitions based on impact and exploitability.

Priority Severity Description Example Vulnerabilities
P1 Critical Complete system compromise RCE, SQL Injection (auth bypass), Account Takeover (no auth), SSRF → internal access
P2 High Significant impact, limited constraints Stored XSS, IDOR (sensitive data), Privilege Escalation
P3 Medium Moderate risk, user interaction needed Reflected XSS, CSRF (non-critical), partial info disclosure
P4 Low Minor impact, hard to exploit Clickjacking, rate limit issues, verbose errors
P5 Informational No real security impact Missing headers, best practice issues
